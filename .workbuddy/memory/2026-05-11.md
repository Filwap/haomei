### 傍晚：修复 Pages 部署失败 (commit cd2d51d)
- **问题**: Cloudflare Pages 部署报错 `The entry-point file at "worker.js" was not found`
- **原因**: `wrangler.toml` 中残留 `main = "worker.js"` 字段，导致 `wrangler deploy` 走 Worker 模式
- **修复**: 删除 `main` 字段，Pages Functions 通过 `functions/` 目录自动识别
- **结果**: 已推送，等待 Cloudflare Pages 自动重新部署

### 晚间：Cloudflare Workers 迁移 + 照片墙修复 (commits 866ff05 → ff2ccb1)

**迁移到 Workers 模式**（4个提交修复部署链）:
1. `866ff05` — 创建标准 Worker 入口 `worker.js`，恢复 wrangler.toml 的 main 字段
2. `4dba2eb` — 移除 rate-limit.js 全局 setInterval（Workers 不允许全局 scope 副作用）

4. `6bf1ccf` — 创建 `.assetsignore` 排除 .git/ 大文件（38MB > 25MB Assets 限制）

**照片墙空白修复** (`ff2ccb1`):
- **问题**: 照片墙区域空白，13张静态照片不显示
- **根因**: `initScrollAnimations()` 给所有 `.gallery-item` 添加 `fade-in` 类（opacity:0），
  但 IntersectionObserver 在某些情况下未能及时触发 `.visible`，导致照片保持透明
- **修复**: HTML 中静态照片默认添加 `visible` 类；JS 跳过已有 visible 的元素
- **状态**: 已推送，等待 Cloudflare Workers Build 自动部署
